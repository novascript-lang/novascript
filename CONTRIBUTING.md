# Contributing to `NovaScript`

Thank you for your interest in contributing to `NovaScript`.

`NovaScript` is an early-stage programming language project. Contributions, feedback, and discussions are extremely valuable as the language and ecosystem evolve.

---

## Ways to Contribute

You can contribute to `NovaScript` in several ways:

* Reporting bugs
* Proposing language improvements
* Improving documentation
* Implementing compiler features
* Improving the runtime
* Writing examples and tutorials

Even small contributions help the project grow.

---

## Getting Started

Before contributing, please:

1. Read the `README.md`
2. Review the language specification in `docs/spec`
3. Check existing issues to avoid duplication

---

## Development Setup

`NovaScript` consists of multiple components:

* `novascript` (compiler and CLI)
* `ns-runtime` (runtime implementation)

Development instructions will evolve as the compiler prototype matures.

---

## Contribution Guidelines

When submitting contributions:

* Keep pull requests focused and small
* Include tests when applicable
* Write clear commit messages
* Follow existing project style conventions

---

## Proposing Language Changes

Language design proposals should:

* Clearly describe the problem being solved
* Explain the proposed syntax or behavior
* Consider impacts on performance and compilation
* Include examples where possible

Large design discussions may happen in GitHub issues before implementation.

---

## Code of Conduct

By participating in the `NovaScript` project you agree to follow the community guidelines described in `CODE_OF_CONDUCT.md`.
